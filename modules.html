Ajax.coffee

Faz a conexão, assíncrona ou síncrona, entre o servidor e o cliente.

Configurações:

autoExec {boolean | default: true}

Indica se o objeto deve ser executado no fim da sua criação.

assinc {boolean | default: true}

Indica se a conexão deve ser assíncrona ou não.

userLoader {boolean | default: true}

Indica se deve ser mostrado um loader para o usuário até a conclusão do carregamento.

contentType {string | default: 'application/x-www-form-urlencoded; charset=UTF-8'}

Indica o tipo de conteúdo da requisição.

url {string | default: null}

Indica a url da requisição.

method {string | default: GET | opções: GET, POST, PUT, DELETE}

Indica o método da requisição.

data {object | default: null}

Indica o conteúdo da requisição.

dataType {string | default: 'json'}

Indica o tipo de retorno esperado após a conclusão da requisição.

Construtor do Ajax {parâmetro: object settings}

{% highlight ruby %} constructor: (settings)-> super @settings = @libSupport.extend true, {}, @settings, settings @debug "Construindo o ajax com as configuracoes: ", @settings @exec() if @settings.autoExec == true {% endhighlight %}

Callbacks:

beforeSend {parâmetros: jqXHR jqXHR, object settings}

Função de retorno pré-requisição que pode ser usada para modificar o objeto jqXHR antes de ser enviado.

complete {parâmetros: jqXHR jqXHR, String textStatus}

Função a ser chamada quando a requisição termina.

error {parâmetros: jqXHR jqXHR, String textStatus, String errorThrown}

Função a ser chamada se a requisição falha.

sucess {parâmetros: PlainObject data, String textStatus, jqXHR jqXHR}

Função a ser chamada se a requisição tiver sucesso.

Métodos:

exec

Executa a requisição javascript.

get_data {retorno: string}

Converte a data em um parametro string.

Animation.coffee

Responsável pelas animações em CSS efetuadas nos DOMElements

Configurações:

animations {array | default: undefined}

Coleção de elementos que receberão os efeitos em ordem.

Contrutor do Animation {parâmetro: object data}

{% highlight ruby %} constructor: (data)-> super @animations = [] @appendAllAnimations(data) if Object.isArray data @appendAnimation(data) if Object.isObject data @createAllAnimations() {% endhighlight %}

Métodos:

appendAllAnimations {parâmetro: array datas}

Método responsável por percorrer os objetos do construtor e enviar um a um para gerar a animação.

appendAnimation {parâmetro: object data}

Responsável por validar e extender as configurações do padrão.

createAllAnimations

Método que percorre a coleção de animações para criar, animação por animação, sendo que cada animação será chamada em cadeia respeitando o tempo de delay da classe CSS.

createAnimation {parâmetros: DOMElement | JQueryElement element, object data, integer indice}

Responsável por criar a animação solicitada.

Core.coffee

Cria uma nova instância da classe Core.

Configurações:

objectId {string | default: null}

Indica o ID de identificação do objeto.

debugIndex {integer | default: 1}

Ele mantém o número de mensagens de depuração impressos pelo objeto.

debugPrefix {string | default: 'joker_Core'}

É o nome do objeto, que será impresso nas mensagens de depuração para identificação.

libSupport {object | default: window.jQuery}

Indica o padrão de ajuda.

className {object | default: 'Joker_Core'}

Define o nome da classe.

settings {object}

É um objeto de controle que contém informações sobre o objeto específico. Todas as informações fornecidas neste objeto podem ser modificadas no momento da instanciação do filho do objeto.

Construtor do Core

{%highlight ruby %} constructor: -> @libSupport = Joker.Core.libSupport @verifyLibSupport() @generateId() {% endhighlight %}

Métodos:

verifyLibSupport {retorno: boolean}

Como o nome diz, verifica se existe biblioteca do jQuery. O método retorna false se existir.

generateId {retorno: string}

Responsável por gerar um id único para o objeto em questão.

destroy {retorno: boolean}

Destrói o objeto e remove as referências do Joker ao Util, limpando a existência do mesmo na memória.

debug {retorno: integer}

Responsável pela verificação e impressão de mensagens de depuração para todos os objetos. Todos os parâmetros passados ​​para os métodos serão impressos na tela. O retorno é o índice da próxima mensagem de depuração.

Debug.coffee

Responsável por imprimir as mensagens de depuração no console. Veja abaixo as configurações e como o sistema de depuração funciona.

Configurações:

console {object | default: window.console}

É responsável por imprimir na tela as mensagens de depuração.

Configuração padrão de depuração.

{% highlight ruby %} paramsDefault: debug : false prefix : 'Joker_Debug' objectId : '' index : 1 messages : [] {% endhighlight %}
debug {boolean | default: false}

Define se a mensagem deve ser impressa ou não.

prefix {string | default: 'Joker_Debug'}

É o nome de identificação que aparecerá quando a mensagem for impressa no console.

objectId {string | default: ''}

Define o id do objeto responsável pela impressão da mensagem no console.

index {integer | default: 1}

Mostra quantas mensagens foram impressas no console. O número é sempre o da próxima mensagem a ser impressa.

messages {array | default: []}

Array contendo as mensagens a serem impressas no console.

Método:

print {parâmetro: object params}

Este método é responsável por imprimir as mensagens no console do navegador. Ele recebe um objeto que contêm os parâmetros para a impressão.

JokerUtils.coffee

O objeto "Utils", contém alguns métodos gerais de ajuda para as classes. Além de armazenamento de referência de instanciação.

Métodos:

uniqid {parâmetro: string prefix, boolean more_entropy | retorno: string}

Gera uma string única para a execução do script.

Exemplos:

{% highlight ruby %} JokerUtils.uniqid(); {% endhighlight %}

Retorno: a30285b160c14

{% highlight ruby %} JokerUtils.uniqid('foo'); {% endhighlight %}

Retorno: fooa30285b1cd361

{% highlight ruby %} JokerUtils.uniqid('bar', true); {% endhighlight %}

Retorno: bara20285b23dfd1.31879087

toString {parâmetro: Mixin | retorno: string}

Converte para string.

addObject {parâmetro: object obj | retorno: object}

Adiciona um objeto para a coleção.

getObject {parâmetro: string id | retorno: object}

Recupera um objeto da coleção.

removeObject {parâmetro: string id | retorno: boolean}

Remove um objeto da coleção.

redirectTo {parâmetro: string url}

Redireciona a página para outra url.

isEmpty {parâmetro: string/Mixin pStrText | retorno: boolean}

Verfica se a string está vazia, incluindo espaços.
